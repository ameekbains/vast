# Contributing to VAST

Thank you for your interest in contributing to VAST! The project is still young and we appreciate all forms of help.

## Reporting issues

Please use the GitHub issue tracker to report bugs or request features. When filing an issue include your system information and steps to reproduce any problems.

## Development setup

1. Install the dependencies listed in the [README](Readme.md#dependencies).
2. Fork the repository and create a feature branch.
3. Make your changes with clear commit messages.
4. Open a pull request describing your changes and any relevant context.

We use the standard GitHub workflow. Pull requests should pass CI and be reviewed before merging.

## Code style

- Follow existing code conventions when possible.
- Write clear comments for complex logic.
- Keep each commit focused on a single change.
